Architecture

    Hermes is the main assistant layer in my personal AI setup.

    Core Design
    - Hermes acts as the primary conversational assistant
    - Personal profile/context files help it understand my goals, preferences, and tone
    - Skills extend its ability to help with specific types of tasks
    - Memory helps it retain useful long-term context
    - Supporting integrations connect it to tools I actually use

    Current Implementations
    - Obsidian integration for note organization and long-term knowledge support
    - Personal “about me” context file for more personalized responses
    - Skill system for reusable task-specific workflows
    - Memory system for persistent context
    - Assistant usage across planning and productivity tasks, including Google Calendar-related help
    - VPS-based environment managed through remote access and CLI workflows

    Technical Learning Areas
    - Working with APIs and tool integrations
    - Learning CLI-based development workflows
    - Using SSH to manage remote systems
    - Building inside a VPS/self-hosted environment
    - Structuring assistant behavior through memory, skills, and context files

    Planned Direction
    - More workflow automation
    - More day-to-day life support
    - More specialized subagents
    - Additional assistant-connected tools and Discord bot workflows
